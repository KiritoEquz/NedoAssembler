## Аргументы:
- Путь к CSV-файлу с исходной программой
- Путь к двоичному файлу с результатом
- Режим тестирования для вывода результата в консоль (--test)

## Команды
- LOAD_CONST,B,C где B - адрес, С - константа
- READ_MEM,B,C где В - адрес, С - адрес
- WRITE_MEM,B,C где В - адрес, С - адрес
- ROR,B,C,D,E где В - адрес, С - адрес, D - смещение, E - адрес

## Функционал
На данный момент ассемблер принимает на вход CSV-файл с выше описанными командами.

В режиме тестирования ассемблер выведет в консоль результат ассемблирования в байтовом формате, как в тесте из спецификации УВМ.

В стандартном режиме ассемблер создаст бинарный файл и запишет в него ассемблированные команды, в консоль будет выведен рамер итогового файла в байтах.
